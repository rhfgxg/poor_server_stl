# 数据库设计及各种数据保存设计

## 用户数据库
为了方便扩展，以后解耦，用户信息分为两部分，一部分是用户的账号基本信息，另一部分是游戏或其他业务的相关信息
保存在不同的表中：
- 用户的账号基本信息保存在 `user` 表中
- 用户的游戏相关信息保存在 `user_game` 表中（不同游戏使用不同的表）
- 用户的网盘相关信息保存在 `user_disk` 表中

不同游戏或业务使用不同数据库
用户注册时，在 `user` 表中插入一条记录
当用户打开某项功能时，在对应的用户信息表中插入一条记录（不关联用户表，只复制用户id）
使用时，在用户校验时，先在 `user` 表中校验，再在对应的用户信息表中校验

更新时，只更新用户表即可

注销时，更新所有表：修改用户表中的状态，删除其他表中的记录

### 用户表（poor_users）
- 用户id（user_id）
- 用户账号（user_account）
- 用户昵称（user_name）
- 密码（password）
- 邮箱（email）
- 手机号（phone_number）
- 身份证号（id_number）
- 头像（avatar）
- 注册日期（registration_date）
- 最后登录日期（last_login_date）
- 状态（status）  // 标记用户是否注销
