# 项目使用技术概览

## 使用语言
1. c++20标准
2. CMake
3. lua 5.3
4. PowerShell
5. BaseShell

## 使用框架与技术
1. Skynet 游戏服务器框架
2. Protobuf 消息序列化
3. GRPC c++ 通信框架（服务器之间，与客户端之间）
4. MySQL 数据库
5. Redis 缓存数据库
6. Git 版本控制
7. GitHub 代码托管

## 项目架构概览
项目采用 Skynet 游戏服务器框架，使用 C++ 作为主要开发语言，Lua 作为脚本语言。项目架构设计如下：
- **服务划分**：项目划分为多个独立的服务模块，包括
    - c++ 网关服务器：负责处理 Skynet 与 C++，客户端与服务器之间的通信与转发，负责与客户端的连接管理
    - c++ 中心服务器：负责管理所有服务器节点，协调各个服务之间的通信
    - c++ 数据库服务器：负责与 MySQL 进行交互，处理数据存储与查询
    - c++ 文件服务器：负责处理文件存储与访问：客户端下载，更新等
    - c++ 登录服务器：负责处理玩家登录与认证
    - c++ redis 缓存服务器：负责与 Redis 集群进行交互，处理比本地存储更低频率的数据访问，本地数据镜像（待开发）
    - lua：逻辑服务器：负责游戏核心逻辑处理，包括玩家管理，游戏玩法，任务系统等
    - lua：匹配服务器：负责处理玩家匹配与组队
    - lua：战斗服务器：负责处理游戏战斗逻辑与状态管理
- **通信机制**：
    - Skynet 服务之间通过消息传递进行通信，使用 Skynet 内置的消息机制
    - C++ 服务器之间通过 GRPC 进行通信，使用 Protobuf 进行消息序列化
    - 客户端与服务器之间通过 GRPC 进行通信，使用 Protobuf 进行消息序列化
    - Skynet 与 C++ 服务器之间通过自定义的 TCP 双向网关进行通信，使用 Protobuf 进行消息序列化
- **数据存储**：
    - 使用 MySQL 作为主要的数据存储方案，存储玩家数据，游戏状态等
    - 使用远程 redis 集群作为缓存数据库，存储高频访问的数据，提高数据访问速度
    - 使用 Redis 作为缓存数据库，存储高频访问的数据，提高数据访问速度
    - 使用本地内存作为直接数据存储，存储短期数据与缓存数据
- **项目管理**：
    - 使用 Git 进行版本控制，管理代码变更与协作开发
    - 使用 GitHub 进行代码托管与项目管理，跟踪问题与任务
